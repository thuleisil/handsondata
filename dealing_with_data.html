Loading the Survey Data

In our study, we’re examining animal species diversity and weights within plots at our research site. The data is stored as a CSV (Comma-Separated Values) file. Each row represents information for a single animal, while the columns include various attributes:

  • record_id: A unique identifier for each observation.
  • month: The month of the observation.
  • day: The day of the observation.
  • year: The year of the observation.
  • plot_id: The ID of a specific experimental plot of land.
  • species_id: A 2-letter code representing the species.
  • sex: The sex of the animal (“M” for male, “F” for female).
  • hindfoot_length: The length of the hindfoot in millimeters.
  • weight: The weight of the animal in grams.
  • genus: The genus of the animal.
  • species: The species of the animal.
  • taxon: The taxonomic classification (e.g., Rodent, Reptile, Bird, Rabbit).
  • plot_type: The type of plot.

CSV File Structure

To work with our survey data, we’re using a CSV file format. This format is commonly used for tabular data storage and exchange. Each row in the CSV file corresponds to an observation, and columns contain different attributes of the observations.

CSV Data Example

Here’s an example of how our survey data might look in the CSV file:

record_id month day year plot_id species_id sex hindfoot_length weight genus species taxon plot_type
1 7 16 1977 2 NL M 32 NA Neotoma albigula Rodent Control
2 7 16 1977 3 NL M 33 NA Neotoma albigula Rodent Control

Importing Data in R

In R, we can import data from CSV files using functions like read.csv() or read.table(). These functions allow us to load the data into R as a data frame, which is a common structure for handling tabular data.

Example:

# Load the data from a CSV file
survey_data <- read.csv("survey_data.csv")

# Display the structure of the data frame
str(survey_data)

Downloading the Data

We’ll use the R function download.file() to download the CSV file containing the survey data from Figshare. We’ll then use read_csv() to load the CSV file’s content into R.

Downloading the CSV File

Within the download.file command, the first entry is a character string representing the source URL (“https://ndownloader.figshare.com/files/2292169”). This URL fetches a CSV file from Figshare.

The text following the comma (“data_raw/portal_data_joined.csv”) is the destination of the downloaded file on your local machine. Ensure you have a folder named data_raw where you’ll save the file.

So, this command downloads a file from Figshare named portal_data_joined.csv and saves it in the existing data_raw folder. 

  download.file(url = "https://ndownloader.figshare.com/files/2292169",
              destfile = "data_raw/portal_data_joined.csv")

Reading the Data into R

The downloaded file is now on your machine, but R hasn’t yet loaded the data from the file into memory. To accomplish this, we’ll use the read_csv() function from the tidyverse package.

Packages in R provide additional functions beyond the base R functions, such as round(), sqrt(), or c(). read_csv() is a function from the tidyverse package, while read.csv() is from base R. The spelling of the function determines which one you use.

Before using a package, you need to install it using the install.packages() function. Install tidyverse by typing install.packages(“tidyverse”) in the console. To load the package for use, simply type library(tidyverse). 

  ## load the tidyverse packages, incl. dplyr
  
library(tidyverse)

What is the Tidyverse?

The Tidyverse is a collection of R packages that provide tools for data manipulation, exploration, and visualization.

It follows the principles of tidy data, aiming to make data analysis more intuitive and efficient.

Key packages include dplyr, tidyr, ggplot2, and more.

Tidy Data Principles

Tidy data follows three principles:

  • Columns: Each variable has a column.
  • Rows: Each observation has a row.
  • Tables: Each type of observational unit has a table.

This structure makes data manipulation and analysis easier and more structured.

Understanding Tidy Data

Tidy data follows a structured format that makes analysis and visualization easier.

Let’s explore examples of untidy data and how to transform it into tidy format.

Example: Untidy Data

Consider this untidy data: 

    
      | Name     | Math | Science | Language |
      | -------- | ---- | ------- | -------- |
      | Alice    | 95   | 82      | 89       |
      | Bob      | 78   | 90      | 75       |
      | Charlie  | 85   | 88      | 92       |

Columns contain multiple variables (Subjects: Math, Science, Language).

Transforming to Tidy Data

To make the data tidy, we can use the pivot_longer() function from the tidyr package.

It will transform the columns into a “key-value” format. 

  tidy_data <- pivot_longer(original_data, cols = -c(Name), names_to = "Subject", values_to = "Score")

Using head() and tail()

The head() function returns the extracted data.

Example: surveys_sample <- head(surveys, 100)

This allows experimenting with computations on a subset before applying to the whole data set.

Similarly, use tail() to extract the last few lines of the data set.

Using head(), tail(), and view()

Useful for trying complex computations on a subset before applying to the whole data set.
